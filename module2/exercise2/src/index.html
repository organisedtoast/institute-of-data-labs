II. Box model and common properties

I set the width to 200 — felt clever, precise,
then padding and border said, “Add us twice!”
I set a tidy width — two hundred on the nose,
Declared, “This box is perfect!” — that’s how the story goes.
Then padding strolled in softly, border joined the scene,
And suddenly my “perfect” box was bursting at the seam.

I whispered, “Box-sizing: border-box,” like casting ancient lore,
The layout stopped exploding… but then I added more.
A margin here, a margin there, a nudge to make it neat,
And somehow now the footer lives halfway up the street.
Flexbox tried to help me — “Relax, I’ll align!”
But justify-content and I can’t read the same sign.
I center things horizontally, vertically I plead,
The box just shrugs and wanders off with reckless flexbox speed.

I add a shadow, rounded corners, gradients for some flair,
Now it looks like modern art — but why’s it over there?
Inspect tool open, twenty divs deep in the tree,
One rogue inherited style is mocking me.

III. Positioning

I set it to absolute — and it vanished with attitude.
I set it to relative — and it moved, but only emotionally.
I'm absolutely positioned inside a relative container.
I'm a sticky positioned floating CSS progress monitor.

IV. Styling text

I chose font-family “DM Sans” refined and sleek,
The browser shrugged and gave me sans-serif all week.
I set font-weight bold 700 in might,
It still looks slightly wrong — but technically right?

VI. calc()

I wrote calc(100% - 20px) feeling smug and wise,
Like maths and style had formed a grand alliance.
The layout shifted left — then down — then back again,
Turns out I forgot the spaces… CSS said “nope” once more, my friend.
